LE JAVASCRIPT

Javascript a été créé en 1995 par Brendan Eich

Le JavaScript : pour quoi faire ?

Pour l'instant dans le domaine du web, vous connaissez HTML (pour la structure et le contenu du document web) et le CSS (pour la mise en forme du document web). Ces 2 langages permettent uniquement de faire des pages web statiques c'est-à-dire "figées" une fois pour toute par le concepteur de la page (si on excepte quelques animations possibles en CSS) Contrairement à HTML et CSS, JavaScript est un langage de programmation.

Comme en python, on retrouvera donc les constructions élémentaires de la programmation comme les notions de variables et de type, d'isntructions conditionnelles, de boucles, de tableau etc...

Grâce à du code Javascript, on va pouvoir manipuler les documents web et donc rendre les pages web dynamiques : les pages web (contenu et mise en forme) peuvent évoluer et interagir avec les actions du visiteur de la page

Le Javascript est un langage côté client : c'est le navigateur web (Firefox, Chrome, etc...) qui va interpréter le code javascript et modifier en conséquence l'affichage obtenu.

Pincipes de la programmation événementielle

La programmation événementielle, consiste à déclencher l'exécution d'une fonction lorsqu'un évènement sur l'élément cible apparaît.

On parle d'abonnement de la fonction à l'élément pour l'évenement. La fonction attachée à un ́evenement est appelee "fonction gestionnaire d'évenement" ou "fonction d'écoute" ou encore fonction "event listener"

Voici quelques exemples d'événements :

  • l'utilisateur a cliqué avec le bouton droit (événement) à tel endroit de la page (élément cible)
  • l'utilisateur a enfoncé (événement) telle touche du clavier (élément cible)
  • la fenêtre (élément cible) a fini de se charger (événement)
  • un téléchargement (élément cible) est terminé (événement)....
  • ...................

Vous trouverez ici la (longue) liste des évènements qu'on peut gérer en javascript.

On peut ainsi créer des interactions avec le visiteur dans une page web. En programmation événementielle, le déroulement du programme n'est plus séquentiel comme ce qu'on a vu jusqu'à présent en python, mais il est déterminé par les différents événements extérieurs qui peuvent apparaître.
